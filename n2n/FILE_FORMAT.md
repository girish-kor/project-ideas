# FILE_FORMAT.md — N2N Toolz Project File Format Specification

---

## 1. File Extensions

| Extension | Description |
|---|---|
| `.n2n` | Project directory bundle (folder, not archive) |
| `.n2nflow` | Single workflow graph JSON |
| `.n2nenv` | Environment variable file (dotenv format) |
| `.n2nplugin` | Plugin package reference file |

---

## 2. Project Directory Layout

```
my-project.n2n/
├── project.json              ← Project metadata and settings
├── workflows/
│   ├── main.n2nflow          ← Main workflow (auto-created)
│   ├── auth.n2nflow          ← Additional workflows
│   └── helpers.n2nflow
├── env/
│   ├── .env.dev              ← Development env vars
│   ├── .env.prod             ← Production env vars (gitignored)
│   └── .env.example          ← Template (committed to VCS)
├── assets/
│   └── [any files used in workflows]
├── plugins.json              ← Plugin references for this project
└── .n2nignore                ← Files excluded from export
```

## 6. .env.dev / .env.prod Format

Standard dotenv format with N2N Toolz type annotations in comments:

```bash
# N2N Toolz Environment Variables
# Generated by N2N Toolz v1.0.0

# @type string @required true
DATABASE_URL=mysql://root:password@localhost:3306/myapp

# @type string @required true @secret true
JWT_SECRET=your-jwt-secret-here

# @type number @required false
PORT=3000

# @type string @required false
NODE_ENV=development
```

## 7. .n2nignore Format

```
# N2N Toolz ignore file (same syntax as .gitignore)

# Exclude production secrets from export
env/.env.prod

# Exclude test workflows
workflows/test-*.n2nflow

# Exclude temp files
assets/temp/
*.tmp
```

## 8. File Versioning

All JSON files include a `$schema` URI and `version` field.

**Migration Strategy:**

```typescript
class FileMigrator {
  migrate(json: unknown, fromVersion: string, toVersion: string): unknown {
    const migrations = getMigrations(fromVersion, toVersion);
    return migrations.reduce((data, migration) => migration.up(data), json);
  }
}

// Example migration: v1.0.0 → v1.1.0
const migration_1_0_to_1_1 = {
  up: (data: WorkflowJSON) => ({
    ...data,
    nodes: data.nodes.map(n => ({
      ...n,
      executionOptions: n.executionOptions ?? DEFAULT_EXECUTION_OPTIONS,
    })),
  }),
};
```

## 9. Export Archive Format (ZIP)

When exporting as ZIP, structure is:

```
project-name-export-2026-03-10.zip
├── src/
│   └── [generated .n2nflow and project.json]
├── dist/
│   ├── index.js
│   ├── routes/
│   ├── workflows/
│   └── lib/
├── package.json
├── .env.example
└── README.md                 ← Auto-generated from project description
```
