## 1. Deployment Modes

| Mode | Description | Trigger |
|---|---|---|
| **Local** | Copy built files to a local path | Deploy → Deploy to Local |
| **Dev Server** | Spawn Node.js child process with hot-reload | Deploy → Start Dev Server |
| **SSH / SCP** | Push build to remote server via SSH | Deploy → Deploy to SSH Target |
| **Docker** | Build Docker image and run container | Deploy → Deploy to Docker |
| **AWS Lambda** | Package as Lambda zip + deploy | Deploy → Cloud → AWS Lambda |
| **GCP Cloud Run** | Containerize + deploy to Cloud Run | Deploy → Cloud → GCP Cloud Run |
| **Custom** | User-defined shell script | Deploy → Custom Endpoint |

## 2. Dev Server

### Start

```
UI: Deploy → Start Dev Server
OR
Generated code: npm run dev
```

### Process Management

```typescript
class DevServerManager {
  private proc: ChildProcess | null = null;
  private port: number;
  private watcher: FSWatcher | null = null;

  async start(distPath: string, port = 3000): Promise<void> {
    await this.buildFirst(distPath);
    this.proc = spawn('node', ['index.js'], {
      cwd: distPath,
      env: { ...safeEnv, PORT: String(port) },
      stdio: 'pipe',
    });
    this.watchForChanges(distPath);
  }

  private watchForChanges(distPath: string): void {
    // Rebuilds on workflow save → restarts server
    this.watcher = watch(distPath, { recursive: true }, async () => {
      await this.restart(distPath);
    });
  }
}
```

## 3. SSH / SCP Deployment

### Target Configuration

```json
{
  "name":     "Production Server",
  "host":     "192.168.1.100",
  "port":     22,
  "username": "deploy",
  "authType": "key",
  "keyPath":  "~/.ssh/id_rsa",
  "remotePath": "/var/www/myapp",
  "preCommand":  "pm2 stop myapp",
  "postCommand": "npm ci --omit=dev && pm2 start index.js --name myapp"
}
```

### Deploy Flow

```
[Build locally → dist/]
        ↓
[SSH connect]
        ↓
[SCP upload: dist/ → remotePath]
        ↓
[Run preCommand via SSH]
        ↓
[Restart app]
        ↓
[Run postCommand via SSH]
        ↓
[Verify: curl health endpoint]
```

## 4. Docker Deployment

### Local Docker Build + Run

```bash
# Generated by N2N Toolz
docker build -t myapp:latest .
docker run -p 3000:3000 --env-file .env.prod myapp:latest
```

### Docker Compose (optional, auto-generated)

```yaml
# docker-compose.yml — Generated by N2N Toolz
version: '3.9'
services:
  app:
    build: .
    ports:
      - "3000:3000"
    env_file:
      - .env.prod
    restart: unless-stopped
    depends_on:
      - db
  db:
    image: mysql:8
    environment:
      MYSQL_ROOT_PASSWORD: ${DB_ROOT_PASS}
      MYSQL_DATABASE: myapp
    volumes:
      - db_data:/var/lib/mysql

volumes:
  db_data:
```

## 5. AWS Lambda Deployment

### Requirements
- AWS CLI configured (`aws configure`)
- Lambda function already created OR auto-create enabled

### Generated Lambda Handler

```javascript
// dist/lambda.js — Generated wrapper
import { createServer, proxy } from 'aws-serverless-express';
import app from './index.js';

const server = createServer(app);
export const handler = (event, context) => proxy(server, event, context);
```

### Deploy Script (auto-generated)

```bash
#!/bin/bash
# deploy-lambda.sh — Generated by N2N Toolz

FUNCTION_NAME="my-n2n-app"
REGION="ap-south-1"

echo "📦 Zipping dist..."
cd dist && zip -r ../lambda.zip . && cd ..

echo "🚀 Deploying to Lambda..."
aws lambda update-function-code \
  --function-name $FUNCTION_NAME \
  --zip-file fileb://lambda.zip \
  --region $REGION

echo "✅ Deployed!"
```

## 6. GCP Cloud Run Deployment

```bash
# deploy-cloudrun.sh — Generated by N2N Toolz
PROJECT_ID="my-gcp-project"
SERVICE_NAME="my-n2n-app"
REGION="asia-south1"

echo "🐳 Building container..."
gcloud builds submit --tag gcr.io/$PROJECT_ID/$SERVICE_NAME

echo "🚀 Deploying to Cloud Run..."
gcloud run deploy $SERVICE_NAME \
  --image gcr.io/$PROJECT_ID/$SERVICE_NAME \
  --region $REGION \
  --platform managed \
  --allow-unauthenticated \
  --set-env-vars "NODE_ENV=production"

echo "✅ Deployed!"
```
