# EXPORT.md — N2N Toolz Export System

---

## 1. Export Targets

| Target | Output | Command (post-export) |
|---|---|---|
| Node.js ESM | `dist/index.js` | `node dist/index.js` |
| Node.js CJS | `dist/index.cjs` | `node dist/index.cjs` |
| TypeScript | `dist/src/*.ts` | `tsc && node dist/index.js` |
| Docker | `dist/` + `Dockerfile` | `docker build -t app . && docker run app` |
| ZIP Bundle | `project-name.zip` | Extract + `npm install && node index.js` |
| JSON Graph | `workflow.n2nflow` | Import into another N2N Toolz project |

## 2. Export Pipeline

```
WorkflowJSON
     │
     ▼
[1. GraphValidator]
     │ ← throws if invalid (missing config, broken edges)
     ▼
[2. DependencyResolver]
     │ ← maps node types → npm packages
     ▼
[3. FileStructurePlanner]
     │ ← decides which files to create based on trigger types
     ▼
[4. CodeGenEngine]           ← per workflow
     │
     ▼
[5. ImportCollector]
     │
     ▼
[6. CodeEmitter (recast)]
     │
     ▼
[7. Prettier Formatter]
     │
     ▼
[8. EntryPointBuilder]       ← generates index.js that wires all routes/workflows
     │
     ▼
[9. PackageJsonBuilder]
     │
     ▼
[10. EnvFileBuilder]         ← .env.example from all secret config fields
     │
     ▼
[11. FileWriter]             ← writes to outputDir
     │
     ▼
[12. PostBuildHook]          ← optional user-defined build script
```

## 3. File Structure Planner Rules

| Condition | Output |
|---|---|
| Workflow has HTTP Trigger node | `routes/[workflow-name].js` |
| Workflow has Cron Trigger node | `jobs/[workflow-name].js` |
| Workflow has Manual Trigger node | `workflows/[workflow-name].js` (exported function) |
| Workflow has Event Trigger node | `events/[workflow-name].js` |
| Multiple HTTP trigger workflows | `routes/` directory with one file per workflow |
| Sub-workflow nodes | `lib/[sub-workflow-name].js` (imported by parent) |

## 5. .env.example Builder

Collects all `secret: true` config fields across all nodes in all workflows:

```javascript
function buildEnvExample(workflows: WorkflowJSON[]): string {
  const envVars: Map<string, { description: string; required: boolean }> = new Map();

  for (const wf of workflows) {
    for (const node of wf.nodes) {
      const def = NodeRegistry.getInstance().get(node.type);
      for (const [key, field] of Object.entries(def.configSchema)) {
        if (field.secret) {
          const envKey = `${node.type.toUpperCase().replace(/-/g, '_')}_${key.toUpperCase()}`;
          envVars.set(envKey, {
            description: field.label,
            required:    field.required ?? false,
          });
        }
      }
    }
  }

  return Array.from(envVars.entries())
    .map(([k, v]) => `# ${v.description}${v.required ? ' (REQUIRED)' : ''}\n${k}=`)
    .join('\n\n');
}
```

## 9. Incremental Export

Only regenerates files for workflows that have changed since last export.

```typescript
class IncrementalExporter {
  private hashes: Map<string, string> = new Map();  // workflowId → content hash

  async export(workflows: WorkflowJSON[], options: ExportOptions): Promise<ExportResult> {
    const changed = workflows.filter(wf => {
      const hash = hashJSON(wf);
      const isDiff = this.hashes.get(wf.id) !== hash;
      if (isDiff) this.hashes.set(wf.id, hash);
      return isDiff;
    });

    const files = await this.codeGenEngine.generate(changed, options);
    await this.fileWriter.write(files, options.outputDir);
    return { generated: files.length, skipped: workflows.length - changed.length };
  }
}
```
